# 跨平台通知功能测试指南

## 概述

通知功能测试需要真机设备和推送服务配置,主要通过手动测试完成。本文档提供详细的测试步骤和验证方法。

## 前提条件

### Android 设备
- ✅ Android 真机或模拟器 (推荐真机)
- ✅ Google Play Services 已安装
- ✅ Firebase Cloud Messaging (FCM) 已配置
- ✅ 应用已安装并授予通知权限

### iOS 设备
- ✅ iOS 真机 (模拟器不支持推送通知)
- ✅ Apple Push Notification Service (APNs) 已配置
- ✅ 应用已安装并授予通知权限
- ✅ 开发者证书和 Provisioning Profile 正确配置

### 后端配置
- ✅ Firebase Admin SDK 已配置
- ✅ APNs 证书已上传到 Firebase
- ✅ 通知服务正常运行

## 测试场景

### 场景 1: 计划邀请通知

**目标**: 验证用户收到计划邀请通知

**测试步骤**:

1. **准备工作**
   - 用户 A 在 Android 设备上登录
   - 用户 B 在 iOS 设备上登录
   - 确保两个设备都已注册推送令牌

2. **发送邀请**
   - 用户 A 在 Android 上创建对赌计划
   - 用户 A 邀请用户 B

3. **验证通知**
   - ✅ 用户 B 的 iOS 设备收到通知
   - ✅ 通知标题: "新的对赌邀请"
   - ✅ 通知内容: "用户A 邀请你参加减重挑战"
   - ✅ 通知包含计划详情

4. **点击通知**
   - 点击通知
   - ✅ 应用打开并跳转到计划详情页
   - ✅ 显示完整的计划信息
   - ✅ 可以接受或拒绝邀请

**预期结果**:
- ✅ 通知在 5 秒内送达
- ✅ 通知内容正确
- ✅ 点击跳转正确

## 常见问题

### Q1: Android 收不到通知

**可能原因**:
1. FCM 配置错误
2. 设备令牌未注册
3. Google Play Services 未安装
4. 应用被系统杀死

**排查步骤**:
```bash
# 查看 FCM 日志
adb logcat | grep FCM

# 检查设备令牌
adb shell dumpsys activity service com.google.android.gms.gcm.GcmService

# 检查应用进程
adb shell ps | grep com.weightloss.betting
```

### Q2: iOS 收不到通知

**可能原因**:
1. APNs 证书配置错误
2. 设备令牌未注册
3. 应用未授予通知权限
4. 使用模拟器 (不支持推送)

**排查步骤**:
1. 检查 Xcode Console 日志
2. 验证 APNs 证书有效期
3. 确认使用真机测试
4. 检查通知权限设置

### Q3: 通知延迟过长

**可能原因**:
1. 网络延迟
2. 推送服务限流
3. 设备省电模式
4. 后端处理慢

**排查步骤**:
1. 检查网络连接
2. 查看 Firebase Console 发送记录
3. 检查后端日志
4. 优化后端通知发送逻辑

### Q4: 点击通知无法跳转

**可能原因**:
1. Deep Link 配置错误
2. 路由逻辑错误
3. 应用未运行

**排查步骤**:
1. 检查通知 payload 中的跳转参数
2. 验证 Deep Link 配置
3. 调试路由逻辑
4. 测试应用在不同状态下的跳转
